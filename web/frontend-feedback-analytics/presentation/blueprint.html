The Blueprint

building safe A.I

Overview

The purpose of this blueprint is to provide a repeatable process that can be used to build safe A.I. using B.C gov on-premises servers.

Who is this for?

Project owners, developers, and stakeholders who are looking to build safe A.I. or integrate A.I. into their existing systems.

Considerations for Building or Using A.I

Scenario Recommendation
Cost-effectiveness and speed are priorities Use an off-the-shelf solution for faster implementation and lower cost (data privacy is not a concern)
Data is highly sensitive or unique Build a custom A.I solution tailored to your specific data security and use case needs
Not sure if A.I is the right solution A.I is not always ideal; assess whether it provides substantial improvements in the specific context

Operational Cost

Cost Type Description
Development Costs Upfront costs such as data acquisition, hiring AI engineers, infrastructure (cloud services, servers), software licenses, and integration of Human-in-the-Loop (HITL) systems.
Operational Costs Ongoing costs including cloud computing power, storage, human expert involvement, data updates, and model re-training.
Monitoring and Maintenance Costs related to system audits, human review, system refinement, and maintaining compliance.
Scalability Cost structure for scaling up, which includes handling increased data, higher AI model complexity, and greater human oversight.

Team Structure for a custom solution

  • AI Engineers, MLOps, Data Scientists: For building and improving the AI models, managing the data pipelines, cleaning data, and ensuring quality input to the AI models, modelling performance.
  • Performance Engineers Optimize data pre-processing, model training and inference for better performance for a givem hardware (CPU).
  • Openshift / Kubernetes Admins: For managing the on-premises servers and ensuring the AI system is running smoothly.
  • Human Experts/Domain Specialists: Who provide input into AI training, monitor outputs and performance for continuous improvement.
  • UX/UI Designers: To create interfaces that allow non-technical users to understand AI decisions and inputs.
  • Product Managers: Oversee system functionality and interaction with clients, ensuring the AI stays aligned with user needs.

Technical Overview

  • Start with using a RAG (Retrieval Augmented Generation) model for your AI system.
  • Index your data using a suitable vector database
  • Handy to use Object storage or similar for storing large data
  • Use analytics tools to monitor, track and improve your AI system
  • Implement MLOps to streamline the A.I workflow for better orchestration of your AI system
  • Traceability and Auditing Mechanism Design a system that logs every decision made by the AI, linking it to the original source (documents, data points).
  • Active Learning Implement active learning to improve the AI model over time. This involves using human feedback to correct and refine the AI's decisions.

System Requirements

  • Data Management System:A robust database structure for storing, updating, and processing the data the AI requires. Ensure traceability is built into the database itself (e.g., versioned data, references to original sources).
  • Human Interaction System: A web-based or software-based interface that allows easy interaction between the AI system and human experts. Include a platform for feedback, review, and approval of AI decisions.
  • Cloud/Open-Shift Infrastructure: Cloud-based systems may be best for speed and perhaps scalability, while highly sensitive data, cost savings and having full control over the A.I pipeline could require using on-premises servers.
  • Security and Compliance: Ensure adherence to data security standards and that both AI and human components maintain audit logs for all actions.

Process for Continous improvement

  • Feedback Loop: Create a clear feedback mechanism where human experts can input their insights on why a decision was right or wrong. Design a process for systematically collecting this feedback to refine the AI model. Ensure there is a version control system in place to track changes over time in AI logic and human interventions.
  • Model Retraining: Schedule regular model retraining, leveraging human input for correction and updating datasets to ensure the AI stays relevant and accurate.
  • Performance Audits: Regular audits to compare AI decisions with human outcomes. This helps refine the HITL balance and ensures continuous learning.
  • Tooling for Traceability: Implement tools to track AI performance, including accuracy metrics, data sources, and links to human interventions.

Risk Management

  • Bias and Fairness: Highlight potential risks of bias in AI models and the role humans play in ensuring fair, unbiased outcomes.
  • Data Privacy: Ensure all data used is compliant with local regulations, and human oversight guarantees sensitive decisions are not fully automated.
  • Ethics and Accountability: Define accountability, ensuring that there is a clear process for investigating and addressing erroneous AI decisions, with human experts taking final responsibility.

Roadmap for Implementation

    • Short-Term (0-6 months):

    • Index your data and set up a basic RAG pipeline
    • Setup tracking to track the performance of the AI system
    • Develop user interfaces for human interaction and audit tracking.
    • Deploy the system on on-premises servers
    • Incorporate MLOps to automate the RAG pipeline, retrieve analytics and other orchestration tasks.

    • Medium-Term (6-12 months):

    • Implement active learning to improve the AI model over time
    • Setup annotation tools for human feedback to correct and refine the AI's decisions
    • Schedule regular model retraining and performance audits
    • Implement High performance computing for faster processing for both training and inference of the AI model
