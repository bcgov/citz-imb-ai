RAG Pipeline

  • Feed the query to sentence transformer
  • The output is fed to NEO4j (vector search)
  • Top K results returned (Top 10)
  • Rerank the topK (optional)
  • Create a prompt which includes Query stack + topk + latest query
  • Feed to gen A.I
  • Get the response
  • Return the response to query

How data is stored in Neo4j

Graph Database and vector store

Data storage Neo4j Integration Advanced Querying
We use a graph database to store the data. Neo4J is leveraged for both the graph database and the vector store. Neo4J enables advanced queries, such as clustering the data and finding communities.
Vector store is utilized for storing embeddings. These tasks are more efficient and easier to implement using a graph database.

Human in the Loop

To improve the A.I model we need to annotate and format the data properly. After the data is annotated we can use it to train the different models.

Embedding Adaptors

If the top sources are not accurate we can retrain the embedding model based on human feedback.

Annotating the data (NER)

For improving our retrieval and enhancing our result we are using an A.I technique called NER (Named Entity Recognition) to annotate the data. this is done by using a tool called Diffgram.
